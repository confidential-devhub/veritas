# Azure

Veritas pulls the `osc-dm-verity-image` from the Red Hat registry,
verifies its signature with cosign, and extracts pre-computed PCR
values from `measurements.json` inside the image.

These PCR values are scraped from a vTPM during UKI podvm boot
(using `measure-image.sh` from coco-podvm-scripts). Veritas does
not recompute them, it reads them directly from the image.

Supports TDX and SNP. The PCR values are the same for both, only
the RVPS key prefix differs (`tdx_` vs `snp_`).

## Usage

```
# TDX (latest dm-verity image)
veritas --platform azure --tee tdx \
  --authfile pull-secret.json \
  --initdata initdata.toml

# SNP
veritas --platform azure --tee snp \
  --authfile pull-secret.json \
  --initdata initdata.toml

# Specific OSC operator versions (merged into one RVPS ConfigMap)
veritas --platform azure --tee tdx \
  --osc-version 1.11.0 \
  --osc-version 1.11.1 \
  --authfile pull-secret.json \
  --initdata initdata.toml
```

## Version discovery

The dm-verity image is versioned by OSC operator release (e.g.
1.11.0, 1.11.1), not by OCP version. When `--osc-version` is not
provided, veritas defaults to `latest`.

Different OSC versions may produce different PCR values (e.g.
pcr09 and pcr12 differ between 1.11.0 and 1.11.1). Use multiple
`--osc-version` flags to merge them into a single RVPS ConfigMap.
